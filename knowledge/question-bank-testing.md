# 测试工程师题库

> 按考察维度分类，每个问题附带追问链。面试官根据候选人经验水平选择。

---

## 测试设计

### Q1: 测试用例设计方法
**问题**：你平时怎么设计测试用例？有哪些常用的设计方法？
**追问链**：
- 等价类划分和边界值分析怎么结合使用？举个例子。
- 判定表法和正交实验法分别在什么场景下使用？
- 场景法和错误推测法的优缺点是什么？
- 给你一个登录功能，你会设计哪些测试用例？（考察全面性）

### Q2: 测试策略与流程
**问题**：你在项目中怎么制定测试策略？测试流程是怎样的？
**追问链**：
- 冒烟测试、回归测试、集成测试分别在什么阶段做？
- 测试左移和测试右移是什么概念？你在实践中怎么落地？
- 需求评审阶段测试应该关注什么？
- 如果项目时间紧张，你会怎么裁剪测试范围？

### Q3: Bug 管理
**问题**：你发现一个 Bug 后的完整处理流程是什么？
**追问链**：
- Bug 报告应该包含哪些关键信息？
- Bug 的严重程度和优先级怎么划分？两者有什么区别？
- 开发说"不是 Bug"或者"无法复现"，你怎么处理？
- 线上出了漏测的 Bug，你怎么做复盘和改进？

---

## 自动化测试

### Q4: UI 自动化测试
**问题**：你做过 UI 自动化测试吗？用的什么框架？
**追问链**：
- Selenium 的元素定位方式有哪些？哪种最稳定？
- 页面对象模型（POM）是什么？为什么要这样设计？
- 自动化测试中怎么处理等待、弹窗、iframe？
- 自动化测试用例的维护成本高怎么解决？

### Q5: 自动化框架设计
**问题**：如果让你从零搭建一个自动化测试框架，你会怎么设计？
**追问链**：
- 数据驱动和关键字驱动的区别是什么？各自优缺点？
- pytest 的 fixture 机制怎么用？conftest.py 的作用？
- 测试报告怎么生成？Allure 用过吗？
- 怎么把自动化测试集成到 CI/CD 流程中？

## 性能测试

### Q6: 性能测试基础
**问题**：你做过性能测试吗？关注哪些指标？
**追问链**：
- 并发数、TPS、响应时间三者的关系是什么？
- 什么是吞吐量拐点？怎么确定系统的最大承载能力？
- JMeter 的线程组怎么配置？怎么做参数化？
- 压测结果怎么分析？怎么定位性能瓶颈？

### Q7: 性能测试进阶
**问题**：性能测试发现 TPS 上不去，你会从哪些方面排查？
**追问链**：
- 服务器资源监控看哪些指标？（CPU、内存、磁盘 IO、网络）
- 数据库慢查询怎么定位和优化？
- 全链路压测和单接口压测有什么区别？怎么做全链路压测？
- 稳定性测试（耐久测试）怎么做？关注什么？

---

## 接口测试

### Q8: API 测试方法
**问题**：你怎么做接口测试？用过哪些工具？
**追问链**：
- RESTful API 测试要验证哪些方面？（状态码、字段、边界值、鉴权）
- Postman 和 RestAssured 各适合什么场景？
- 接口之间有依赖关系时怎么处理？（比如先登录再调业务接口）
- 接口测试的断言一般包含哪些内容？

### Q9: 接口自动化与契约测试
**问题**：接口自动化测试框架怎么设计？怎么提高覆盖率？
**追问链**：
- 接口测试数据怎么准备？怎么清理？
- 异步接口怎么测试？轮询还是回调？
- 契约测试（Contract Testing）是什么？和集成测试有什么区别？
- 微服务架构下的接口测试策略是什么样的？

## 评价维度

| 维度 | 优秀表现 | 不合格表现 |
|------|---------|-----------|
| 测试设计 | 用例覆盖全面，方法运用灵活 | 只会凭经验写用例，方法单一 |
| 自动化 | 能独立搭建框架，集成 CI/CD | 只会录制回放，不理解框架设计 |
| 性能测试 | 能独立设计压测方案，定位瓶颈 | 只会用工具跑脚本，不会分析结果 |
| 接口测试 | 体系化测试，关注安全和边界 | 只验证正向流程，缺少异常场景 |
